
# static
- 叫静态，**可以修饰成员变量、成员方法**


# 成员变量
- 访问自己类中的类变量可以**省略类名**，访问别的类的静态变量不能省略类名


**成员变量按照有无static修饰，分为两种**
- **类变量**(静态变量)：有static修饰，在内存中只会存在一份，会被类和类的实例共享
	- **不推荐使用实例访问以及通过实例修改**
- **实例变量**：无static修饰，只能实例对象访问


**代码**
Test.java
```java
public class Test {
    public static void main(String[] args) {
        // static修饰成员变量的用法

        // 静态变量访问方式
        // 方式1： 类名.静态变量访问
        Student.name = "袁华";
        System.out.println(Student.name); // 袁华

        // 方式2: 对象.静态变量访问(不推荐)
        Student s1 = new Student();
        s1.name = "马冬梅";
        System.out.println(s1.name); // 马冬梅


        // 实例变量访问方式
        // 对象.实例变量
        s1.age = 12;
        System.out.println(s1.age); // 12

    }
}
```
Student.java
```java
public class Student {
    // 静态变量 有static修饰，属于类持有，在内存中只加载一次，类本身和它的实例都能访问
    static String name;

    // 实例变量，无static修饰 只能实例访问
    int age;
}
```

# 成员方法
- 访问自己类中的类方法可以**省略类名**，访问别的类的静态方法不能省略类名

**成员方法按照有无static修饰，分为两种**
- **类方法**(静态方法)：有static修饰
- **实例方法**：无static修饰，只能实例对象访问

**代码**
Test.java
```java
public class Test {

    public static void main(String[] args) {
        // 类方法访问方式
        // 方式1: 类名.方法名
        Student.printHello();

        // 方式2 对象.方法名（不推荐）
        Student s1 = new Student();
        s1.printHello();

        // 实例方法访问
        // 对象.方法名
        Student s2 = new Student();
        s2.printScore();
    }
}
```

Student.java
```java
public class Student {
    double score;

    // 静态方法 通过static修饰
    // 当前类和他的所有实例对象可以访问
    public static void printHello() {
        System.out.println("hello");
    }

    // 实例方法 没有static修饰
    // 只能实例对象访问
    public void printScore() {
        System.out.println("分数：" + score);
    }
}
```

## 使用场景

- 类方法最常见的应用场景就是做**工具类**，能提高代码复用
- 工具类没有创建对象的需求，**建议将工具类的构造器私有化**

**工具类为什么不用实例方法**
- 实例方法需要创建对象调用，工具类只是为了调用方法，如果创建对象会占用内存浪费资源

**代码**
- 如果多处都需要用到验证码这个功能只需要调用VerifyCodeUtils即可
- 不需要每个用到的地方都重新手写一下实现
```java
public class Test {

    public static void main(String[] args) {
      VerifyCodeUtils.code(4);
    }
}
```

VerifyCodeUtils.java
```java
public class VerifyCodeUtils {

	// 私有化工具类构造器 外界可以直接通过类调用 不需要实例对象
	// 因为类会默认有一个无参构造器，如果要私有那么需要手写出来用private修饰
	private VerifyCodeUtils() {}
	
	// 生成验证码的方法
	public static String code(int len) {
		String code = "xxx"
		// 执行生成验证码的逻辑
		
		return code
	}
}
```

### 注意事项

- 1. 静态方法中只能访问静态变量和方法
- 2. 实例方法中可以访问 实例和静态的 方法和变量
-  this只能在实例方法中使用
```java
public class Test {
    // 静态变量
    public static String name;
    // 实例变量
    public int age;
    // 静态方法
    public static void printName() {
        System.out.println(name);
    }

    public static void main(String[] args) {

    }

    // 1. 静态方法中只能访问静态变量和方法
    public static void testStatic() {
        System.out.println(name); // 可以访问静态变量
        // System.out.println(age); 报错 不能访问实例变量
        // this.age; 报错 静态方法不能有this
    }

    // 2. 实例方法中可以访问 实例和静态的 方法和变量
    // 3. this只能在实例方法中使用
    public void testInstance() {
        System.out.println(name);
        System.out.println(age);
        printName();
        System.out.println(this.age);
    }
}
```
### 代码块
- 代码块是类的5大成分之一(成员变量、构造器、方法、代码块、内部类)


**按照有无static代码块分为两种**

**静态代码块**
- **格式**： static {}
- **特点**：类加载时自动执行，由于类只会加载一次，所以静态代码块也**只会执行一次**
- **作用**：完成类的初始化，例如对类变量的初始化赋值
```java
public class CodeTest1 {
    static ArrayList<String> names = new ArrayList<>();

    public static void main(String[] args) {
        System.out.println("main执行了");
    }

    // 静态代码块
    // 用的比较少 了解即可 方便看别人的源码
    static {
        names.add("hhhh"); // 初始化
        System.out.println("static代码块与类一起加载，比main先执行");
    }
}

```
**实例代码块**
- **格式**： {}
- **特点**：每次创建对象实例时，执行实例代码块，**并在构造器之前执行**
- **作用**：和构造器一样，都是用来完成对象的初始化的，例如对 实例变量进行初始化赋值
```java
public class CodeTest2 {
    public static void main(String[] args) {
        // 每次实例新的对象都会调用一次实例代码块和构造器
        CodeTest2 t1 = new CodeTest2();
        CodeTest2 t2 = new CodeTest2();
    }
    
    public CodeTest2() {
        System.out.println("我在实例代码之后执行");
    }

    // 实例代码块
    // 每次创建对象时都会执行一次，且在构造器之前执行
    // 理解即可
    {
        System.out.println("实例代码块：我在实例代码之前执行");
    }
}
```

### 单例模式

- 确保一个类只有一个对象
- 单例模式有很多种


**饿汉单例**
- 拿到对象的时候，对象早已创建好了
```java
public class Single {
    // 2. 创建一个静态变量来存放对象
    // 确保只有一个类的实例
    // 这个静态变量要私有化 防止外部可以更改
    // 外部使用的时候对象已经创建好了(饿汉单例)
    private  static Single single = new Single();

    // 1. 私有化构造器
    private Single () {}

    // 3. 定一个一个静态方法 供外部初始化
    public static Single getInstance() {
        return single;
    }
}
```

**懒汉单例**
- 只有拿对象时，才开始创建对象

```java
public class Single2 {
    // 2. 定义一个静态方法记住类的一个唯一对象
    // 因为是懒汉单例所以不会在类创建的时候就初始化对象
    private static Single2 single2;

    // 1. 私有化构造器
    private Single2() {}

    // 3. 定义一个静态方法 供外部初始化
  public static Single2 getInstance() {
        if (single2 == null) {
            single2 = new Single2();
        }

        return  single2;
  }
}
```

# 继承
- 使用`extends`关键字完成继承
- A类称为父类(基类、超类)
- B类称为子类(派生类)

```java
public class B extends A { }
```


**特点**
- **子类中**能继承父类的**非私有成员**(成员变量、成员方法)
- 一旦继承后，子类的对象是由子类和父类等多张设计图共同完成的，但是子类对象能访问什么，还是要看权限(public/private)
	- 子类对象中其实是有父类或者其他设计图的变量、方法的，只不过访问要看是否有权限

**代码**
Test.java
```java
public class Test {
    public static void main(String[] args) {
        // b对象只能访问A类、B类中有权限的
        B b = new B();
        b.afn2();
        b.bfn2();
        System.out.println(b.b1);
        System.out.println(b.a1);
    }
}
```
A.java
```java
public class A {
    public  int a1;
    private int a2;
    private void afn1() {}
    public void afn2() {}
}
```
B.java
```java
public class B extends A {
    public int b1;
    private void bfn1() {
        // 只能访问父类中非私有的成员变量、成员方法
        afn2();
        System.out.println(a1);
    }
    public void bfn2() {}
}

```

## 权限修饰符
- 就是用来限制类的成员(成员变量、成员方法、构造器、代码块等等)能够被访问的范围

- **private**：只能本类中访问
- **缺省**(不写):同一个包的类、本类中访问
- **protected**:同一个包的类、本类、子孙类中访问
- **public**:任意位置访问
```java
public class Fu  {
    // 1. private修饰的 只能在当前类中访问
    private void printMethod() {
        System.out.println("private");
    }

    // 2. 缺省(没有)修饰符的 ：同一个包的类、本类中访问
    void  method() {
        System.out.println("缺省");
    }

    // 3. protected修饰的：同一个包的类、本类、子孙类中访问
    protected void  protectedMethod() {
        System.out.println("protected");
    }
    
    // 4. public修饰的任意位置都能访问
    public void publicMethod() {
        System.out.println("public");
    }
}
```

- 但是支持多层继承(A extends B, B extends C, C extends E)

## 方法重写
- 当子类觉得父类中的某个方法不好用，或者无法满足自己的需求时，子类可以重新写一个，**方法名称、参数列表一样**的方法，去覆盖父类的这个方法
- 重写后，方法的访问，java会遵循就近原则

**注意**
- 使用**Override注解**，它可以指定java编译器，检查我们方法重写的格式是否正确，代码的可读性也会更好
- 子类重写父类方法时，访问权限必须大于等于父类该方法的权限(public>protected>缺省)
- 重写的方法返回值类型，必须与被重写的方法的返回值类型一样，或者范围更小
- 私有方法、静态方法不能被重写

**代码**
Cat.java
```java
public class Cat extends Anima {
    public static void main(String[] args) {
        Cat c = new Cat();
        // run方法重写过 java遵循就近原则访问
        // 因为Cat中有所以会访问 Cat的run
        c.run();
    }
    
    @Override
    // 1. 重写的run方法 权限必须大于或等于 父类的run的权限
    // 2. 重写的run的返回值类型， 必须和父类的run的返回值一样，或者比它的返回值范围更小
    void run() {
        System.out.println("父类的run不好 我重新写");
    }
}

```

Anima.java
```java
public class Anima {
    void run() {
        System.out.println("跑起来");
    }
}
```

## 子类访问其他成员的特点

**1. 在子类方法中访问其他成员(成员变量、成员方法)，是按照就近原则的**
- 先子类局部范围找
- 然后子类成员范围找
- 然后父类成员范围找，找不到就报错

**2. 如果子类强行要使用父类的成员，使用super关键字调用**

```java
public class Test {
    public static void main(String[] args) {
        Son s = new Son();
        s.showName();
    }
}

class Son extends Father {
    String name = "son";

    public void showName() {
        String name = "showName";

        // 访问当前方法的name
        System.out.println(name); // showName
        // 访问子类的name
        System.out.println(this.name); // son
        // 访问父类的name
        System.out.println(super.name); // father
    }
}

class Father {
    String name = "father";

}

```

## 子类构造器特点

- 子类的全部构造器，在调用之前，都会先调用父类的**无参构造器**，然后在执行自己的构造器

**原因**
- 默认情况下，子类的全部构造器的第一行代码是 **super()** ，这行代码写不写都存在，作用是调用父类的无参构造器
- 如果父类没有无参构造器(假如只有有参构造器)，则必须在子类构造器的第一行手写 **super(参数)**，去调用父类的有参构造器

## this(xxx)调用兄弟构造器
- 任意类型的构造器中，可以通过**this(xxx)** 去调用该类的其他构造器
- 注意 **super(xxx) 和 this(xxx) 不能共存** ，且它们必须在构造器的第一行
- 作用之一是：可以暴露更少的参数，使用默认值

```java
  
public class Test {  
    public static void main(String[] args) {  
        // 1  
        Student s1 = new Student("张三",18,"北大");  
  
        // 假设想要创建一个对象 school是默认值  
        Student s2 = new Student("李四", 21);  
        System.out.println(s2); // Student{name='李四', age=21, school='清华'}  
    }  
}
```

```java
public class Student {
    private String name;
    private int age;
    private String school;

    public Student() {
    }

    public Student(String name, int age) {
        // this(xx) 会去调用它的兄弟构造器
        // 去匹配符合参数列表的兄弟构造器
        this(name,age,"清华");
    }

    public Student(String name, int age, String school) {
        this.name = name;
        this.age = age;
        this.school = school;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public String getSchool() {
        return school;
    }

    public void setSchool(String school) {
        this.school = school;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", school='" + school + '\'' +
                '}';
    }
}

```

# 多态
- 多态是在**继承/实现**情况下的一种现象，表现为：对象多态、行为(方法)多态

**多态前提**
- 有**继承/实现**关系
	- cat和dog都是继承animal
	- 否则cat和dog就没发使用animal的类型表示了
- 存在父类引用子类对象
	- cat和dog类型都是animal 但是他们指向的是子类
- 存在方法重写
	- cat和dog都是重写了cry方法
	- 不重写的话cry就会使animal的cry了

**注意**
- 多态是对象、行为的多态，java中的属性(成员变量)是没有多态的


**代码**
```java
public class Test {
    public static void main(String[] args) {
        // 对象多态
        // 对象多态：父类引用指向子类对象
        // animal可以是cat 也可以是 dog这就是对象的多态
        // cat和dog类型都是animal 但是他们指向的是子类

        // 行为多态
        // 行为多态：同一个方法调用表现出不同行为
        // cat和dog的叫是不同的 但是都是叫这个行为 这就是行为多态
        Animal cat = new Cat();
        cat.cry();


        Animal dog = new Dog();
        // 方法调用：编译看左，运行看右什么意思
        // 在编译阶段 看=左边 看Animal 是否有cry这个方法 否则报错
        // 运行时 是去=右边 也就是Dog中找cry执行
        dog.cry();
        
        // 变量：编译看左，运行还是看左
        // 无论是编译还是运行都看=左边的类型 也就是Animal 去这个类中找name
        // 说明变量是没有多态性的
        System.out.println(dog.name); // Animal
    }

}

```

```java
public class Dog extends Animal {
	public String name = "Dog";
	
    @Override
    public void cry() {
        System.out.println("狗：汪汪汪叫");
    }
}

```

```java
public class Cat extends Animal{
	public String name = "Cat";
	
    @Override
    public void cry() {
        System.out.println("喵咪-喵喵喵叫");
    }
}
```

```java
public class Animal {
    public String name = "Animal";

    public void cry() {
        System.out.println("动物会叫");
    }
}
```

## 优点

**1. 在多态形式下，右边的对象是解耦的，更便于拓展和维护**
```java
public class Test {
    public static void main(String[] args) {
        // 1. 在多态形式下，右边的对象是解耦的，更便于拓展和维护
        // 假如页面上有狗和猫两类
        // 默认是猫  现在点击狗了 那么只需要更改new Cat() -> new Dog()
        // 调用的方法哪里就不用改变了 如果后面还有别的代码也是关于多态的 也不用更改
        Animal a1 = new Cat();
        a1.cry();
    }
}
```

**2.定义方法时，使用父类型作为行参类型，可以接受一切子类对象，扩展性更强，更便利**
```java
public class Test {
    public static void main(String[] args) {
    // 2. 定义方法时，使用父类型作为行参类型，可以接受一切子类对象，扩展性更强，更便利
    // go方法可以传入animal的所有子类 这样就不用额外的在go中做判断了
    public void go(Animal a) {
        a.cry();
    }
}
```

## 缺点
- 多态下不能调子类的独有功能
- 解决方案 使用强制转换
```java
public class Test {
    public static void main(String[] args) {
        // 假如Dog有一个方法是lookDoor animal是没有的
        // 那么不能调用lookDoor方法
        Animal a1 = new Dog();
        a1.cry();
        
        // a1.lookDoor() // 报错
    }
}
```

## 类型转换
- **自动类型转换**：父类 变量名 = new 子类();
	- 就是上面多态的写法 `eg: People p = new Teacher();`
- **强制类型转换**：子类 变量名 = 子类 父类变量
	- `eg: Teacher t = (Teacher) p;`

**注意事项**
- 存在继承/实现关系的两个类在编译阶段就可以进行强制类型转换，编译阶段不会报错
- 运行时，如果发现转换的对象和真实对象不一致，就会报错
- 所以Java建议在强制转换前，**判断当前对象的真实类型**，在进行强制转换
	- 使用 **instanceof**关键字

**代码**
```java
public class Test {
    public static void main(String[] args) {

        Animal a1 = new Cat();
        a1.cry();

        // 如果想要调用子类独有的方法 则需要强制转换成子类
        Cat c1 = (Cat) a1;
        c1.eatFish();
    }

    public static void go(Animal a) {
        // 注意事项
        // 只要有继承或则实现关系的两个类就可以强制转换
        // 编译阶段不会报错，但是运行阶段可能会有类型转换异常
        // Dog d1 = (Dog) a1; 把猫强制转换成狗了 运行时发现有问题就会报错
        // 所以java建议强制类型转换前，先对类型进行真实性判断
        if (a instanceof Dog) {
            Dog d1 = (Dog) a;
            d1.lookDoor();
        } else if (a instanceof Cat) {
            Cat c1 = (Cat) a;
            c1.eatFish();
        }
    }
}
```

# final

**final关键字是最终的意思，可以修饰(类、方法、变量)**

- **修饰类**：该类被称为最终类，特点是不能被继承
- **修饰方法**：该方法为最终方法，特点是不能被重写
- **修饰变量**：该变量只能被赋值一次


**修饰变量注意事项**
- final修饰基本类型的变量，变量存储的数据不能被改变
- final修饰引用类型的变量，变量存储的地址不能被改变，但是地址所指向的内容是可以改变的

**代码**

```java
public class FinalDemo1 {
    // 4. 修饰静态成员变量
    // static final修饰的成员变量叫做常量，值只有一个，且不能更改
    // 常量在声明的时候就要赋值 因为类在加载的时候需要SCHOOL_NAME有值
    // 常量名称建议大写
    public static  final String  SCHOOL_NAME = "哇哈哈";

    // 或者在静态代码块中赋值
    // 因为类和静态代码块是一起加载的
    public static final String SCHOOL_NAME2;
    static {
        SCHOOL_NAME2 = "农夫山泉";
    }

    // 5. 修饰实例成员变量
    // 无意义 这会导致所有的实例的name一样 且不能修改
    final String name = "哈哈哈";
    public static void main(String[] args) {
        /*
            java的变量有两种
            1. 局部变量：方法内部、形参、for循环变量、构造器的变量都是局部变量
            2. 成员变量：静态成员变量和实例成员变量
         */

        // 3. 修饰局部变量
        // 有且仅能赋值一次
        final int a = 12;
        // a = 13; // 报错 因为只能赋值一次
    }
}

//final class A { }
```

## 常量
- 使用 static final修饰的成员变量就被称为常量

**优点**
- 代码可读性更好、可维护性也更好
- 程序编译后，常量会被“宏替换”
	- 出现常量的地方会被全部替换成其记住的字面量，这样可以保证使用常量和直接使用字面量的性能是一样的
	- 也就是说 当编译成可执行代码的时候 变量全部会被替换成字面量

# 抽象类

- 使用 **abstract**关键字修饰类、成员方法
- abstract修饰类，这个类就是抽象类
- 修饰方法，这个方法就是抽象方法

**语法**
```
修饰符 abstract class 类名 {
	修饰符 abstract 返回值类型 方法名称 (形参列表);
}
```

**代码**
```java
public abstract class A {
    // 定义抽象方法
    // 抽象方法只有方法签名 没有方法体
    public abstract void go();
    public  abstract  void run();
}
```

**注意事项、特点**
- 抽象类中不一定有抽象方法，抽象方法只能存在抽象类中
- 类该有的成员(成员变量、方法、构造器)抽象类都可以有
- 抽象类**不能创建对象**，仅作为一种特殊的父类，让子类继承并实现
- 一个类继承抽象类，必须重写完该抽象类的全部抽象方法，否则这个类也必须定义成抽象类
	- 言外之意就是 抽象类可以继承另一个抽象类


## 使用场景

- 父类知道每个子类都要做某个行为，但每个子类要做的情况不一样，父类就把这个行为定义成抽象方法，交给子类去重写实现，**这样设计抽象类，是为了更好的支持多态**

## JDK8后新增的三种方法
- 新增这些方法是**增强了接口的能力**，方便项目的拓展和维护
- 开发中业务中使用的概率很低很低
- 学习只是为了后面看源码不懵逼

```java
// 新增的这些 实现类可以不重写
public interface A {
    // 1. 默认方法(普通方法,实例方法): 必须使用default修饰
    // 默认会用public修饰
    // public default void run() {}
    // 必须用接口的实现类的对象来调用，因为接口是不能new 对象的
     default void run() {
        // 可以调用当前接口内部的私有方法
         go();
         go1();
         System.out.println("接口A");
     }

     // 2. 私有方法(私有的实例方法),JDK9开始有的
     // 只能当前接口内部的  默认方法或者私有方法来调用
    private void go() {
        System.out.println("接口A的私有方法go");
    }
    private void go1() {
        // 可以调用当前接口内部的私有方法
        go();
        System.out.println("接口A的私有方法go1");
    }

    // 3. 静态方法
    // 默认使用public修饰 public可以省略
    // 接口的静态方法必须使用接口名本身调用
    static void fn() {
        System.out.println("fn接口静态方法");
    }
}
```

## 接口注意事项
- 下面说的默认方法就是实例方法
- 方法签名就是抽象方法的签名

 **一个接口继承多个接口，如果多个接口中存在方法签名冲突，则此时不支持多继承**
```java
// A1和A2都有run 但是他们的返回值不一样这时候就是方法签名冲突了
// 如果他们的返回值都一样 那么就不冲突
interface A1 {
    void run();
}
interface A2 {
    String  run();
}
interface A3 extends A1,A2 {} // 报错签名冲突 不能多继承
```

 **一个类实现多个接口，如果多个接口中存在方法签名冲突，则此时不支持多实现**
 ```java
 class AImpl implements A1,A2 {} // 不能多实现 因为A1和A2的方法签名冲突了
 ```


 **一个类继承了父类，又同时实现了接口，父类中和接口中有同名的默认方法，实现类会优先用父类的**
```java
interface A2 {
	// 默认方法
    default void go() {
        System.out.println("我是接口的go");
    }
}

class B {
    public void go() {
        System.out.println("我是父类的go");
    }
}

class AImpl extends B implements A2 {
}

public class Test {  
    public static void main(String[] args) {  
        AImpl a = new AImpl();  
        a.go();  // 我是父类的go
    }  
}
```

**一个类实现了多个接口，多个接口中存在同名的默认方法，可以不冲突，这个类重写该方法即可**
```java
interface A1 {
    default void  go() {
        System.out.println("我是A的go");
    }
}

interface A2 {
    default void  go() {
        System.out.println("我是实现类的go");
    }
}

class AImpl implements A1,A2 {
    // 多实现的接口如果有相同的默认方法
    // 实现类需要重写该方法 否则报错
    @Override
    public void go() {
        A1.super.go();
    }
}
```

# 枚举

**语法**
```
修饰符 enum 枚举名称 {
	名称1, 名称2, ...;
	其他成员...
}
```

**注意**
- 枚举类的第一行只能罗列一些名称，这些名称都是**常量**，且每个常量记住的都是枚举类的一个对象(每个常量就是这个枚举的每个实例，且每个实例有且只能有一个，也就是每个常量只会初始化一次)
- 枚举类的构造器是私有的(写不写都只能是私有的)，因此，枚举类不能被创建对象
- 枚举都是最终类，不能被继承
- 枚举类中，从第二行开始，可以定义类的其他各种成员
- 编译器为枚举类新增了几个方法，且枚举类都是继承java.lang.Enum类的，从Enum类也会继承到一些方法

**代码**
```java
public class Test {
    public static void main(String[] args) {
        A a1 = A.X;
        System.out.println(a1); // X

        // values获取A的所有常量到数组中
        A[] as = A.values();
        for (int i = 0; i < as.length; i++) {
            System.out.println(as[i]);
        }

        // valueOf获取A的某个常量
        A x = A.valueOf("X");
        System.out.println(x);// X
        System.out.println(x == a1);// true

        // 获取某个常量的索引
        System.out.println(a1.ordinal());// 0
    }
}


public enum A {
    // 第一行，只能写一些合法的标识符(名称)，多个名称用逗号隔开
    // 这些名称，本质上是常量，每个常量都会记住枚举类的一个对象
    X, Y, Z;
    String name;
    int age;
}
```

## 使用场景

- 用来表示一组信息，然后作为参数进行传递
```java
public enum Constant {  
    DOWN, UP, HALF_UP;  
}

public class Test {
    public static void main(String[] args) {
        System.out.println(handleData(100.22, Constant.DOWN));
    }
    // 枚举作信息标准和分类：很优雅，参数值受到约束，是信息标志和分类的理想方案

    // 也可以使用常量，但是常量就没法约束分类的值了
    // 不过使用常量的话性能会比枚举好，因为常量在编译后会进行宏替换，枚举还要实例对象
    // 所以就要在可读性和性能作取舍了
    public static double handleData(double number, Constant flag) {
        // switch模式下 可以不写枚举名 Constant.DOWN,直接写常量值就行
        switch (flag) {
            case DOWN:
                // 向下取整
                return Math.floor(number);
            case UP:
                // 向上取整
                return Math.ceil(number);
            case HALF_UP:
                // 四舍五入
                return Math.round(number);
            default:
                return number;
        }
    }
}

```

# 泛型

- 定义类、接口、方法时，**同时声明一个或多个类型变量，称为泛型类**、泛型接口、泛型方法，它们统称为泛型
- 泛型提供了在编译阶段约束所能操作的数据类型，并自动进行检查的能力
- 这样可以避免强制类型转换，及其它可能出现的异常。
- 泛型的本质：**把具体的数据类型作为参数传给类型变量**

**注意： 类型变量建议用大写英文字母，常用的有：E、T、K、V等**

```java
public class Test {
    public static void main(String[] args) {
	    // <String> 就是泛型 和ts差不多
        ArrayList<String> arr = new ArrayList<>();
        arr.add("zs");
        arr.add("ls");
    }
}

```

## 泛型类
-  只要类能接收泛型 就称为泛型类
```
修饰符 class 类名<类型变量1, 类型变量2, ...> {}
```

```java
// 泛型类
// 只要类能接收泛型 就称为泛型类
public class MyArrayList<E> {
    public boolean add(E e) {
        return  true;
    }
}
```

## 泛型接口
- 只要接口能接收泛型 就称为泛型接口
```
修饰符 class 类名<类型变量1, 类型变量2, ...> {}
```

```java
// 泛型接口
// 只要接口能接收泛型 就称为泛型接口
public interface Date<T> {  
    void add(T t);  
    void delete(T t);  
}
```

## 泛型方法
-  只要方法能接收泛型 就称为泛型方法
```
修饰符 <类型变量1, 类型变量2, ...> 返回值类型 方法名(形参列表){}
```

```java
public static <T> void go() {}

// 返回类型也是泛型
public static <T> T fn(T[] arr) {  
    return  arr[0];  
}
```

## 通配符
- `?`表示，可以在使用泛型的时候代表一切类型
- 注意是使用泛型，不是定义泛型的时候

```java
public class Test2 {
    public static void main(String[] args) {
        // 通配符
        ArrayList<BMW> bmws = new ArrayList<>();
        bmws.add(new BMW());
        bmws.add(new BMW());
        go(bmws);

        ArrayList<Audi> aduis = new ArrayList<>();
        aduis.add(new Audi());
        aduis.add(new Audi());
        go(aduis);


    }

    // 场景：go方法需要接收任意品牌的汽车集合（如BMW集合、Audi集合）
    // 但ArrayList<BMW>和ArrayList<Audi>是两种不同类型，无法用同一泛型接收
    // 通配符：? 表示，可以在使用泛型的时候(注意是不是定义泛型的时候),代表一切类型
    public static void go(ArrayList<?> cars) {}
}

class Car {}
class BMW extends Car {}
class Audi extends Car {}
```

## 泛型的上下限
- 泛型上限：`? extends 类型` 
	- ？必须是类型的子类或者当前类
- 泛型下限：`? super 类型`
	- ？必须是当前类型或者当前类型的父类

```java
public class Test2 {
    public static void main(String[] args) {
        // 通配符
		...通配符的代码
		
        ArrayList<Dog> dogs = new ArrayList<>();
        dogs.add(new Dog());
        dogs.add(new Dog());
       //  go(dogs); 不能使用 类型不对

    }

    // 场景：go方法只希望接收汽车家族的集合（Car及其子类），拒绝其他类型如Dog
    // 解决方案：使用泛型上限 ? extends Car，表示只能是Car或Car的子类
    public static void go(ArrayList<? extends Car> cars) {}
}
...通配符的代码

class Dog {}
```

## 泛型擦除
- 泛型是工作在编译阶段的，一旦程序编译成class文件，class文件中就不不存在泛型了，这就是泛型擦除

## 注意事项
- 泛型不能直接支持**基本类型数据类型**，只能支持引用数据类型
