# Object
- Object类是Java中所有类的祖宗类
- 因此Java中所有类的实例都可以直接使用Object中提供的一些方法

### 常见方法

| 方法名                               | 说明           |
| --------------------------------- | ------------ |
| `public String toString()`        | 返回对象的地址      |
| `public boolean equals(Object o)` | 判断两个对象地址是否相同 |
## 示例
```java
public class Test {
    public static void main(String[] args) {
        Student s1 = new Student("小明",18);
        // 1.toString方法
        // toString默认返回对象的地址，没有什么意义，开发中其实是想看内容，所以toString是为了让子类重写
        // toString()可以省略 默认println就是调用toString
        // System.out.println(s1.toString());
        System.out.println(s1);

        // 2. equals方法
        // equals方法默认是比较两个对象的地址是否一样
        // 比较两个对象的地址是否一样可以直接使用"=="比较，完全没必要用equals
        // 所以 Object提供的equals的意义是为了让子类重写，以便自己定制比较规则(按内容比较)
        Student t1 = new Student("小明",18);
        Student t2 = new Student("小明",18);
        System.out.println(t1.equals(t2)); // true 因为子类重写了
        System.out.println(t1 == t2);// false
    }

}

public class Student {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }

    @Override
    public boolean equals(Object o) {
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return age == student.age && Objects.equals(name, student.name);
    }
}

```

# Objects
- Objects是一个工具类，提供了很多操作对象的静态方法
- 这个就是类似js中Reflect和对象的关系
- 所以工具类都有一些特征：不能创建对象，构造器都私有，提供静态方法供调用

## 常见方法

| 方法名                                                | 说明             |
| -------------------------------------------------- | -------------- |
| `public static boolean equals(Object a, Object b)` | 先做非空判断，在比较两个对象 |
| `public static boolean isNull(Object o)`           | 判断对象是否是null    |
| `public static boolean nonNull(Object o)`          | 判断对象是否不是null   |

## 代码

```java
public class Test {
    public static void main(String[] args) {
	    // 这里的Student使用的是上面Object章节的类
        Student s1 = null;
        Student s2 = new Student();
        // 如果s1是null那么比较的时候就会报错 空指针异常
        // System.out.println(s1.equals(s2));

        // 使用Objects提供的equals 则不会 因为内部做了非空判断
        // 今后比较两个对象时 建议使用Objects.equals
        // 本质上Objects.equals内部调用的还是Student重写后的equals 只不过加了非空判断
        System.out.println(Objects.equals(s1, s2)); // false 不报错


        System.out.println(Objects.isNull(s1)); // true
        System.out.println(Objects.nonNull(s1)); // false
    }
}

```

# 包装类
- 包装类就是把基本类型的数据包装成对象
- 由于泛型和集合都不支持基本数据类型，因为万物皆对象，因此包装类在集合和泛型中大量且必须使用

| 基本数据类型  | 对于的包装类    |
| ------- | --------- |
| byte    | Byte      |
| short   | Short     |
| int     | Integer   |
| long    | Long      |
| char    | Character |
| float   | Float     |
| double  | Double    |
| boolean | Boolean   |
## 示例
-  下面的示例使用的是int做示范，其余的都是一样的
```java
public class Test {
    public static void main(String[] args) {
        // 把基本类型通过包装类包装成对象

        // 一、手动包装
        // 1. 方式一 构造器 已经废弃 因为valueOf会缓存
        Integer it1 = new Integer(1);
        // 2. 方式二 valueOf方法
        // 这个方法内部其实会创建 -128到127 之间的整数 然后缓存起来
        // 这样在使用的时候就不用每次都new对象了
        // 因为这个数区间比较常用，超过了这些就会重新new
        Integer it2 = Integer.valueOf(1);

        // 二、自动装箱机制：基本类型的数据可以直接变成对象赋值给包装类
        // 等同于 Integer it3 = Integer.valueOf(1);
        Integer it3 = 1;

        // 三、自动拆箱机制：包装类的对象可以直接赋值给基本数据类型
        int it4 = it3;
        System.out.println(it4); // 1
    }
}

```

# StringBuilder

- StringBuilder代表可变字符串对象，相当于是一个容器，它里面装的字符串是可以改变的，就是用来操作字符串的。
- 好处：**StringBuilder**比**String**更适合做字符串的修改操作，效率会更高，代码也会更简洁。

- 如果操作字符串较少，或者不需要操作，以及定义字符串变量，还是建议使用String，如果频繁操作字符串建议使用**StringBuilder**

## 构造器

| 构造器 | 说明 |
| --- | --- |
| public StringBuilder() | 创建一个空白的可变字符串对象，不包含任何内容 |
| public StringBuilder(String str) | 创建一个指定字符串内容的可变字符串对象 |

## 常见方法

| 方法名称                              | 说明                                       |
| --------------------------------- | ---------------------------------------- |
| public StringBuilder append(任意类型) | 添加数据并返回StringBuilder对象本身                 |
| public StringBuilder reverse()    | 将对象的内容反转                                 |
| public int length()               | 返回对象内容长度                                 |
| public String toString()          | 通过toString()就可以实现把StringBuilder转换为String |
## 代码
```java
public class Test {
    public static void main(String[] args) {
        // 创建空的string
        StringBuilder sb = new StringBuilder();
        // 初始化内容的string
        StringBuilder sb2 = new StringBuilder("你好");

        // append方法
        sb.append(1).append("你好啊");
        System.out.println(sb); // 1你好啊

        // 反转内容
        System.out.println(sb.reverse()); // 啊好你1

        // length
        System.out.println(sb.length()); // 4

        // 转成String
        // 为啥还要转成String？
        // StringBuilder是拼接字符串的手段
        // String才是开发的目的
        // 因为开发中很多api接收的值类型是String 加上"" 存字符串会节省内容等原因
        String s1 = sb.toString();

    }
}

```

# StringJoiner

- JDK8开始才有的，跟StringBuilder一样，也是用来操作字符串的，也可以看成是一个容器，创建之后里面的内容是可变的。

- 好处：不仅能提高字符串的操作效率，并且在有些场景下使用它操作字符串，代码会更简洁

## 构造器

| 构造器 | 说明 |
| --- | --- |
| public StringJoiner(间隔符号) | 创建一个StringJoiner对象，指定拼接时的间隔符号 |
| public StringJoiner(间隔符号, 开始符号, 结束符号) | 创建一个StringJoiner对象，指定拼接时的间隔符号、开始符号、结束符号 |

## 方法

| 方法名称                               | 说明                     |
| ---------------------------------- | ---------------------- |
| public StringJoiner **add**(添加的内容) | 添加数据，并返回对象本身           |
| public int **length**()            | 返回长度（字符出现的个数）          |
| public String **toString**()       | 返回一个字符串（该字符串就是拼接之后的结果） |
## 代码
```java
public class Test {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3, 4, 5};
        System.out.println(returnStringArray(arr)); // [1, 2, 3, 4, 5]
    }

    static String returnStringArray(int[] arr) {
        if (arr == null) return null;

        StringJoiner sj = new StringJoiner(", ", "[", "]");
        for (int i = 0; i < arr.length; i++) {
            sj.add(Integer.toString(arr[i]));
        }
        return sj.toString();
    }
}
```

# BigDecimal
- 主要做精确运算的
## 构造器

| 构造器                           | 说明                                                                                                      |
| ----------------------------- | ------------------------------------------------------------------------------------------------------- |
| public BigDecimal(double val) | 将double转成BigDecimal  （**不推荐使用这个**，假如两个double相加超出了double的范围，BigDecimal是可以接住的，但是它解决不了精度丢失的问题，可以解决大数据相加问题） |
| public BigDecimal(String val) | 把String转成BigDecimal，这个可以解决精度丢失                                                                          |
## 常用方法

| 方法名                                                     | 说明                    |
| ------------------------------------------------------- | --------------------- |
| public static BigDecimal **valueOf**(double val)        | 转换一个double成BigDecimal |
| public BigDecimal **add**(BigDecimal b)                 | 加法                    |
| public BigDecimal **subtract**(BigDecimal b)            | 减法                    |
| public BigDecimal **multiply**(BigDecimal b)            | 乘法                    |
| public BigDecimal **divide**(BigDecimal b)              | 除法                    |
| public BigDecimal **divide**(另一个BigDecimal对象，精确几位，舍入模式) | 除法、可以控制精确到小数几位        |
| public double **doubleValue**()                         | 将BigDecimal转换为double  |
## 代码
```java
public class Test {
    public static void main(String[] args) {
        double a = 0.1;
        double b = 0.2;
        System.out.println(a + b); // 0.30000000000000004 精度丢失

        // 解决精度丢失问题
        // 方式A 使用构造器 转换成BigDecimal对象 不过要先把浮点数转成字符串
        BigDecimal a1 = new BigDecimal(Double.toString(a));
        BigDecimal b1 = new BigDecimal(Double.toString(b));
        System.out.println(a1.add(b1)); // 0.3

        // 方式B 阿里巴巴公司更推荐使用valueOf方法包装浮点型整数成为BigDecimal对象
        // valueOf本质内部也是使用构造器的方式封装了一下
        BigDecimal a11 = BigDecimal.valueOf(a);
        BigDecimal b11 = BigDecimal.valueOf(b);
        System.out.println(a11.add(b11)); // 0.3
        
         // divide遇到除不尽报错解决方案
        BigDecimal i = BigDecimal.valueOf(10);
        BigDecimal j = BigDecimal.valueOf(3);

        // double k = i.divide(j)
        // 这样会报错 因为10/3是除不尽的 但是BigDecimal本身就要求是精确的
        // 所以遇到除不尽的可以使用divide的重载方法
        // 声明精确到几位 第三个参数是对 保留几位小数之后的取舍
        double k = i.divide(j,2, RoundingMode.CEILING).doubleValue();
        System.out.println(k); // 3.34 因为是向上取整 是对小数点后面的两位做向上取整
    }
}
```

# JDK8 时间API（java.time）
- 所有时间对象都是不可变的，如果要修改是不会修改原对象的所以要通过一个新的变量承接

## 核心类速查

| 类                   | 操作单位                               |
| :------------------ | :--------------------------------- |
| `LocalDate`         | 年、月、日                              |
| `LocalTime`         | 时、分、秒                              |
| `LocalDateTime`     | 年、月、日 + 时、分、秒                      |
| `ZoneId`            | 时区（如 `ZoneId.of("Asia/Shanghai")`） |
| `ZonedDateTime`     | 带时区的日期时间                           |
| `Instant`           | 时间戳（秒/纳秒，UTC）                      |
| `DateTimeFormatter` | 格式化/解析                             |
| `Duration`          | 时间差（时、分、秒、纳秒）                      |
| `Period`            | 日期差（年、月、日）                         |

## 通用方法规律（5个核心类通用）

| 操作 | 方法模式 | 示例（LocalDate） |
|:---|:---|:---|
| 获取当前 | `now()` | `LocalDate.now()` |
| 手动创建 | `of(...)` | `LocalDate.of(2026, 6, 5)` |
| 字符串解析 | `parse(str)` | `LocalDate.parse("2026-06-05")` |
| 获取字段 | `getXxx()` | `getYear()`、`getDayOfWeek()` |
| 加减 | `plusXxx()` / `minusXxx()` | `plusDays(3)`、`minusMonths(1)` |
| 改值 | `withXxx()` | `withYear(2025)` |
| 比较 | `isBefore()` / `isAfter()` | `date1.isBefore(date2)` |
| 格式化 | `format(formatter)` | `date.format(DateTimeFormatter.ISO_DATE)` |

> 记忆口诀：**创建 of/now/parse，加减 plus/minus，改值用 with，比较 before/after**

## 差异点（易忘/特殊）

| 类 | 特殊方法 |
|:---|:---|
| `LocalDate` | `getDayOfWeek()`、`isLeapYear()`、`toEpochDay()` |
| `LocalTime` | 无 `plusDays` 等日期相关方法 |
| `LocalDateTime` | `toLocalDate()` / `toLocalTime()` 拆分 |
| `ZonedDateTime` | `withZoneSameInstant()` 转换时区 |
| `Instant` | `toEpochMilli()`、`plusSeconds()`、`atZone(zone)` |
| `Duration` | `between(t1, t2)`、`toMinutes()`、`getSeconds()` |
| `Period` | `between(d1, d2)`、`getYears()` / `getMonths()` / `getDays()` |

## 代码

```java
// 创建
LocalDate date = LocalDate.of(2026, 6, 5);
LocalTime time = LocalTime.of(14, 30);
LocalDateTime dt = LocalDateTime.of(date, time);
ZonedDateTime zdt = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));

// 运算
date.plusDays(10);           // 2026-06-15
dt.minusHours(2);            // 2026-06-05 12:30

// 格式化
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm");
// 下面这俩效果一样 一个是formatter时间 一个是时间被formatter
String str = dt.format(formatter);  // "2026-06-05 14:30"
String str = formatter.format(dt);  // "2026-06-05 14:30"

// 时间差
Duration d = Duration.between(time, LocalTime.of(16, 0));
d.toMinutes();               // 90


// 解析字符串时间成对象然后就能参与运算了  
String dateStr = "2026-01-02 12:22:35";  
// 创建一个解析器  
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");  
// 解析成LocalDateTime对象  
// parse也可以不放formatter 但是字符串必须是"2026-01-02T12:22:35" 这种标准时间  
LocalDateTime ldt = LocalDateTime.parse(dateStr,formatter);  
System.out.println(ldt); // 2026-01-02T12:22:35
```
# Arrays

- 用来操作数组的一个工具类。

## Arrays类提供的常见方法

| 方法名                                                                     | 说明               |
| ----------------------------------------------------------------------- | ---------------- |
| public static String **toString**(类型[] arr)                             | 返回数组的内容          |
| public static int[] **copyOfRange**(类型[] arr, 起始索引, 结束索引)               | 拷贝数组（指定范围）       |
| public static **copyOf**(类型[] arr, int newLength)                       | 拷贝数组             |
| public static **setAll**(double[] array, IntToDoubleFunction generator) | 把数组中的原数据改为新数据    |
| public static void **sort**(类型[] arr)                                   | 对数组进行排序（默认是升序排序） |
## 代码
```java
public class ArraysDemo1 {
    public static void main(String[] args) {
        int[] arr1 = {1, 2, 3, 4};

        // 返回数组的内容
        System.out.println(arr1); // [I@6acbcfc0 直接打印是地址
        System.out.println(Arrays.toString(arr1)); // [1, 2, 3, 4]

        // 指定范围拷贝数组
        int[] arr2 = Arrays.copyOfRange(arr1, 0, 3);
        System.out.println(Arrays.toString(arr2)); // [1, 2, 3]

        // 拷贝数组到新数组 且指定长度
        // 如果长度小于原数组会截断拷贝
        // 大于的话后面补0
        int[] arr3 = Arrays.copyOf(arr1, 10);
        System.out.println(Arrays.toString(arr3)); // [1, 2, 3, 4, 0, 0, 0, 0, 0, 0]

        // 修改数组的内容
        double[] scores = {80.22, 90.21, 55.1, 66.32};
        Arrays.setAll(scores, new IntToDoubleFunction() {
            @Override
            public double applyAsDouble(int index) {
                return scores[index] + 10;
            }
        });
        System.out.println(Arrays.toString(scores)); // [90.22, 100.21, 65.1, 76.32]

        // 数组排序(升序)
        int[] sort = {1, 3, 2, 5, 6};
        Arrays.sort(sort);
        System.out.println(Arrays.toString(sort));

    }
}
```


## 自定义排序
- 很多时候数组里面不一定是数组，很多时候是对象

**两种方式**
- 方式一： 让该对象的类实现Comparable接口，然后重写compareTo方法，自己来制定比较规则
- 方式二：使用下面的sort方法，创建Comparator比较器接口的匿名内部类对象，然后自己制定比较规则，（sort方法是重载方法）
	- `public static <T>void sort(T[] arr, Comparator<? super T> c) `





**公共代码**
```java
public class ArraysDemo2 {
    public static void main(String[] args) {
        Student[] students = new Student[3];
        students[0] = new Student("张三",18,'男');
        students[1] = new Student("李四",15,'男');
        students[2] = new Student("小红",22,'女');
```

- **作用**是：用于简化匿名内部类的代码写法

## **格式**
```
(被重写方法的形参列表) -> {
	被重写方法的方法体代码
}
```

**Lambda表达式只能简化函数式接口的匿名内部类**

## 特定类型的引用

### 语法
```
类型::方法
```

### 使用场景
- 如果某个Lambda表达式里只调用一个实例方法，且前面参数列表中的第一个参数是作为方法的主调，后面的所有参数都是作为该实例方法的入参的，则此时就可以只用特定类型的方法引用

### 代码
```java
public class Test3 {
    public static void main(String[] args) {
        String[] names = {"lulu", "Luxi", "Angela", "张三", "Baby", "angel"};

        // 需求对names进行排序且忽略大小写

        // 默认排序的话 不会忽略大小写 不符合预期
        // Arrays.sort(names);

        // 只能自定义排序

        // 如果自己写排序规则很麻烦且复杂
        // 好在内置了有对字符串排序的api可以调用
        Arrays.sort(names, new Comparator<String>() {
            @Override
            public int compare(String o1, String o2) {
                // 使用String提供的实例方法忽略大小写排序
                return o1.compareToIgnoreCase(o2);
            }
        });

        // 使用Lambda进行简化
        Arrays.sort(names, (o1, o2) -> o1.compareToIgnoreCase(o2));


        // 使用特定类型方法引用简化
        // 因为上面Lambda表达式只调用一个实例方法
        // 且参数列表中的第一个参数是作为方法的主调 o1.compareToIgnoreCase
        // 后面的所有参数都作为该实例方法的入参 o1.compareToIgnoreCase(o2)
        // 所以可以使用特定类型方法引用简化
        Arrays.sort(names,String::compareToIgnoreCase);
        
        // [angel, Angela, Baby, lulu, Luxi, 张三]
        System.out.println(Arrays.toString(names));
    }
}
```

## 异常的体系

**Error**：代表的系统级别错误(属于严重问题)，也就是说系统一旦出现问题，sun公司会把这些问封装成Error对象给出来，说白了，Error是给sun公司自己用的，不是给我们程序员用的，因此我们开发人员不用管它。
