# 集合体系结构

分为**Collection**(单列集合)和**Map**(双列集合)
- Collection代表单列集合，每个元素(数据)只包含一个数据
- Map代表双列集合，每个原始包含两个值(键值对)

**集合不支持基本类型，可以使用对应的包装类**

## Collection集合特点

- List系列集合：添加的元素是**有序、可重复、有索引**
	- `ArrayList`、`LinkedList`：有序、可重复、有索引

- Set系列集合：添加的元素是**无序、不重复、无索引**
	- `HashSet`：无序、不重复、无索引
	- `LinkedHashSet`：**有序**、不重复、无索引
	- `TreeSet`：**按照大小默认升序**、不重复、无索引
- Set的常用方法，基本上就是Collection提供的，自己几乎没有额外新增一些常用功能

## Collection常用方法

- Collection是单列集合的祖宗，它规定的方法（功能）是全部单列集合都会继承的。
- ArrayList、LinkedList这些都有下面的方法

| 方法名                                         | 说明                     |
| ------------------------------------------- | ---------------------- |
| `public boolean add(E e)`                   | 把给定的对象添加到当前集合中         |
| `public void clear()`                       | 清空集合中所有的元素             |
| `public boolean remove(E e)`                | 把给定的对象在当前集合中删除         |
| `public boolean contains(Object obj)`       | 判断当前集合中是否包含给定的对象       |
| `public boolean isEmpty()`                  | 判断当前集合是否为空             |
| `public int size()`                         | 返回集合中元素的个数             |
| `public Object[] toArray()`                 | 把集合中的元素，存储到数组中         |
| `boolean addAll(Collection<? extends E> c)` | 将参数集合中的所有元素，逐个添加到当前集合中 |
## Collection的遍历方式

### 迭代器

- 迭代器是用来遍历集合的专用方式（数组没有迭代器），在Java中迭代器的代表是 **Iterator**。
#### Collection集合获取迭代器的方法

| 方法名称 | 说明 |
|----------|------|
| `Iterator<E> iterator()` | 返回集合中的迭代器对象，该迭代器对象默认指向当前集合的第一个元素 |

#### Iterator迭代器中的常用方法

| 方法名称 | 说明 |
|----------|------|
| `boolean hasNext()` | 询问当前位置是否有元素存在，存在返回true，不存在返回false |
| `E next()` | 获取当前位置的元素，并同时将迭代器对象指向下一个元素处 |

#### 代码
```java
public class CollectionDemo1 {
    public static void main(String[] args) {
        // 创建一个Collection ArrayList继承Collection 所以可以使用他的遍历方式
        // 多态写法 类型是父 实际是子类型
        Collection<String> list = new ArrayList<>();
        list.add("周杰伦");
        list.add("王力宏");
        list.add("范冰冰");

        // 得到集合对象的迭代器对象
        Iterator<String> it = list.iterator();

        // 循环迭代器
        while (it.hasNext()) {
            String ele = it.next();
            System.out.println(ele);
        }

    }
}
```

### 增强for循环

- 增强for可以用来遍历**集合或者数组**
- **增强for遍历集合**，本质就是迭代器遍历集合的简化写法

**修改增强for中的变量值不会影响到集合中的元素**

#### 语法
```
for (元素的数据类型  变量名 : 数组或集合) { }
```

#### 代码
```java
public class CollectionDemo2 {
    public static void main(String[] args) {
        // 创建一个Collection ArrayList继承Collection 所以可以使用他的遍历方式
        Collection<String> list = new ArrayList<>();
        list.add("周杰伦");
        list.add("王力宏");
        list.add("范冰冰");

        // 增强for写法遍历集合
        for (String s : list) {
	        s = "11"; // 无效不会修改集合的原内容
            System.out.println(s); // 这里都变成11了因为修改了变量
        }
        
        // 原内容没有被修改
		System.out.println(list); // [周杰伦, 王力宏, 范冰冰]
		
        // 增强for遍历数组
        int[] age = {18, 20, 21};
        for (int i : age) {
            System.out.println(i);
        }

    }
}
```

### Lambda表达式遍历集合

- 得益于JDK 8开始的新技术Lambda表达式，提供了一种更简单、更直接的方式来遍历集合。

 **需要使用Collection的如下方法来完成**

| 方法名称                                               | 说明           |
| -------------------------------------------------- | ------------ |
| `default void forEach(Consumer<? super T> action)` | 结合lambda遍历集合 |
- `Consumer<? super T> action` 这一坨是函数式接口，会使用Lambda来简化所以不用关心`Consumer`

#### 代码
```java
public class CollectionDemo3 {
    public static void main(String[] args) {
        // 创建一个Collection ArrayList继承Collection 所以可以使用他的遍历方式
        Collection<String> list = new ArrayList<>();
        list.add("周杰伦");
        list.add("王力宏");
        list.add("范冰冰");

        // forEach方式配合Lambda简化

        // 1.原始写法（匿名内部类）
        list.forEach(new Consumer<String>() {
            @Override
            public void accept(String s) {
                System.out.println(s);
            }
        });

        // 2.使用Lambda简化匿名内部类
        list.forEach(s -> System.out.println(s));
    }
}

```
### 遍历的区别

**结论**
- 如果有遍历并删除的需求可以使用**迭代器的方式(推荐)**
- 如果是ArrayList带索引的集合，可以使用for循环删除，每次退一步，或者从后面倒着删除并遍历
```java
public class CollectionDemo5 {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        list.add("大先生");
        list.add("董小姐");
        list.add("安河桥");
        list.add("八卦");

        // 需求：遍历到“董小姐”的时候需要删除

        // 1. 使用迭代器遍历
        Iterator<String> it = list.iterator();
        while (it.hasNext()) {
            String name = it.next();
            if (name.contains("董小姐")) {
                // 如果使用迭代器遍历，且用集合的remove删除数据，会报错ConcurrentModificationException
                // list.remove(name);

                // 使用迭代器的remove就不会报错
                it.remove();
            }
        }
        System.out.println(list);


        // 2. 使用增强for
        ArrayList<String> list2 = new ArrayList<>();
        list2.add("大先生");
        list2.add("董小姐");
        list2.add("安河桥");
        list2.add("八卦");

        // 使用增强for(一定会报错，且无法解决)
        // 一定会报错 因为本质上遍历集合内部使用的就是迭代器
        // 但增强for没法获取到迭代器对象所以就没法调用迭代器的remove
        for (String s : list2) {
            if (s.contains("董小姐")) {
                list2.remove(s);
            }
        }
        System.out.println(list2);

        // 3. 使用forEach配合Lambda
        // forEach也不能解决
        ArrayList<String> list3 = new ArrayList<>();
        list3.add("大先生");
        list3.add("董小姐");
        list3.add("安河桥");
        list3.add("八卦");
        list3.forEach(s-> {
            if (s.contains("董小姐")) {
                list3.remove(s);
            }
            System.out.println(s);
        });

    }
}

```

## List集合

**属于Collection的子类，除了拥有父类的功能还有自己的特有功能**

### 特有方法
- List集合因为支持索引，所以多了很多与索引相关的方法

| 方法名称                             | 说明                  |
| -------------------------------- | ------------------- |
| `void add(int index, E element)` | 在此集合中的指定位置插入指定的元素   |
| `E remove(int index)`            | 删除指定索引处的元素，返回被删除的元素 |
| `E set(int index, E element)`    | 修改指定索引处的元素，返回被修改的元素 |
| `E get(int index)`               | 返回指定索引处的元素          |

#### 代码
```java
public class ListDemo1 {
    public static void main(String[] args) {
        List<String> list = new ArrayList();
        list.add("喜羊羊");
        list.add("美羊羊");
        list.add("懒羊羊");

        // add 在集合的指定位置插入元素
        list.add(1, "沸羊羊");
        System.out.println(list); // [喜羊羊, 沸羊羊, 美羊羊, 懒羊羊]

        // 指定位置删除索引
        String removed = list.remove(1);
        System.out.println(removed); // 沸羊羊
        System.out.println(list); // [喜羊羊, 美羊羊, 懒羊羊]

        // 指定位置修改元素 返回被修改的元素
        String updated = list.set(0, "慢羊羊");
        System.out.println(updated); // 喜羊羊
        System.out.println(list); // [慢羊羊, 美羊羊, 懒羊羊]

        // 返回指定索引的元素
        String firstEle = list.get(0);
        System.out.println(firstEle); // 慢羊羊
    }
}

```

### ArrayList底层原理

#### 特点
- 基于数组实现的
- **根据索引查询数据的速度快**：查询数据通过地址值和索引定位，查询任意数据耗时相同
- **删除效率相对较低**：可能需要把后面很多的数据进行前移，注意是相对某集合，不是一定都低
- **添加效率极低**：可能需要把后面很多的数据后移，在添加元素；或者也可能需要进行数组的扩容

#### 底层原理
-  利用无参构造器创建的集合，会在底层创建一个默认长度为0的数组(这个默认数组是一个常量，也就是只要无参构造器创建的集合，都共享这个数组)
- 添加第一个元素时，底层会创建一个**新的**长度为10的数组
- 存满时，会扩容1.5倍然后返回一个**新的数组**(比如添加10个之后数组长度变成15的新数组)
- 如果一次添加多个元素，1.5倍还放不下，则新创建数组的长度以实际为准
	- 假如使用setAll一次添加20个元素，那么**就会重新创建**一个长度为20的数组

#### 应用场景

- **适合**根据索引查询数据，比如根据随机索引取数据(高效)，或者数据量不是很大的时候
- **不合适**数据量大的同时，又要频繁的进行增删操作

### LinkedList

#### 链表
- 链表中的节点是对的对象，在内存中是不连续的，每个节点包含数值和下一个节点的地址

**特点**
- **查询慢**，无论查询那个数据都要从头开始
- **链表增删相对快**，因为只需要更改指针指向即可

#### LinkedList集合底层原理
![[Pasted image 20260706194608.png]]

- 基于**双链表**实现的，每个节点除了指向下一个节点地址，还有一个指向上一个节点的地址
- 双链表是牺牲内存换速度

**特点：** 查询慢，增删相对较快，但对**首尾**元素进行**增删改查**的速度是**极快**的，因为拥有首尾双指针，可以直接定位首尾位置

**所以针对首尾操作频繁的都可以使用LinkedList 比如队列、栈**

#### 方法

**新增了很多首尾操作的特有方法**

| 方法名称                        | 说明               |
| --------------------------- | ---------------- |
| `public void addFirst(E e)` | 在该列表开头插入指定的元素    |
| `public void addLast(E e)`  | 将指定的元素追加到此列表的末尾  |
| `public E getFirst()`       | 返回此列表中的第一个元素     |
| `public E getLast()`        | 返回此列表中的最后一个元素    |
| `public E removeFirst()`    | 从此列表中删除并返回第一个元素  |
| `public E removeLast()`     | 从此列表中删除并返回最后一个元素 |

#### 代码
```java

public class ListDemo4 {
    public static void main(String[] args) {
        LinkedList<String> queue = new LinkedList<>();
        queue.addFirst("张明");
        queue.addFirst("王木");

        // 在该列表开头插入指定的元素
        queue.addFirst("小明");

        // 将指定的元素追加到此列表的末尾
        queue.addLast("王华");

        System.out.println(queue); // [小明, 王木, 张明, 王华]

        System.out.println(queue.getFirst()); // 小明
        System.out.println(queue.getLast()); // 王华
        
    }
}

```

## Set集合

**Set的常用方法，基本上就是Collection提供的，自己几乎没有额外新增一些常用功能**

### 哈希值
- 就是一个**int**类型的**数值**(随机数)，Java中**每个对象**都有一个哈希值
- Java中所有的对象，都可以调用Object类提供的hashCode方法，返回该对象自己的哈希值
- `public int hashCode(): 返回对象的哈希值`

**特点**
- 同一个对象**多次调用**hashCode()方法返回的哈希值是**相同的**
- 不同的对象，他们的hash值**一般不相同**，但也有**可能会相同**

### HashSet

**无序、不重复、无索引**

#### 底层原理
- 基于哈希表实现
- 哈希表是一种增删改查数据，性能都比较好的数据结构

**哈希表**
- JDK8之前，哈希表=数组+链表
- JDK8开始，哈希表=数组+链表+红黑树

#### 对象不重复添加

因为每个对象的hashCode大概率是不一样的，如果两个对象内容一模一样，添加到**HashSet**中，是不会去重的，因为在添加内容的时候**HashSet**集合会首先比较hashCode，然后在比较内容的是否一样

**解决方案**
- 重写对象的`equals`和`hashCode`即可

```java

public class SetDemo1 {
    public static void main(String[] args) {
        Student s1 = new Student("张三", 18);
        Student s2 = new Student("李四", 18);
        Student s3 = new Student("张三", 18);

        Set<Student> set = new HashSet<>();
        set.add(s1);
        set.add(s2);
        set.add(s3);

        System.out.println(set); // [Student{name='张三', age=18}, Student{name='李四', age=18}]
    }
}

public class Student {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public boolean equals(Object o) {
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return age == student.age && Objects.equals(name, student.name);
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

```

### LinkedHashSet

**有序、不重复、无索引**

- 依然是基于哈希表实现的
- 但是，他的每个元素都额外的多了一个双链表的机制记录它前后元素的位置
- 所以他是有序的

### TreeSet

**按照大小默认升序、不重复、无索引**
**底层基于红黑树实现排序，增删改查性能较好**

- 对于数值类型：Integer、Double默认按照数值本身的大小进行升序排序
- 对于字符串类型：默认按照首字符的编号升序排序

#### 存放自定义对象

TreeSet集合存储自定义类型对象时，**必须指定排序规则**，**否则会报错**

**方式一**
- 让自定义的类，实现**Comparable**接口，重写里面的**compareTo**方法来指定比较规则

```java

public class SetDemo2 {
    public static void main(String[] args) {

        // 存储自定义对象必须要自定义排序 否则会报错
        Set<Student> set = new TreeSet<>();
        set.add(new Student("张三", 18));
        set.add(new Student("李四", 31));
        set.add(new Student("王五", 20));
        set.add(new Student("赵六", 31));

        System.out.println(set);
    }
}

public class Student implements Comparable<Student> {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public int compareTo(Student o) {
        // 因为TreeSet底层是基于红黑树储存的
        // 对于相同的值(这里比较的是age)，也就是相同的age那么是不会重复存储的
        //  如果想要相同的age也存储 需要这么做
        // 也就是说需要自己手动去声明 无论大于还是等于都返回1 这样就就能重复存储了
        return this.age - o.age >= 0 ? 1 : -1 ;
    }
    
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

```

**方式二**
- 通过调用TreeSet集合有参构造器，可以设置Comparator对象(比较器对象，用于指定比较规则)
- `public TreeSet(Comparator<? super E> comparator)`
```java
public class SetDemo3 {
```

## Collection总结

**项目中无法分清就用Arraylist和HashSet**

如果希望记住元素的添加顺序，需要存储重复的元素，**又要频繁的根据索引查询数据**？
- **用ArrayList集合（有序、可重复、有索引），底层基于数组的。（常用）**

如果希望记住元素的添加顺序，**且增加首尾数据的情况较多**？
- **用LinkedList集合（有序、可重复、有索引），底层基于双链表实现的。**

如果不在意元素顺序，**也没有重复元素需要存储，只希望增加改查都快**？
- **用HashSet集合（无序、不重复、无索引），底层基于哈希表实现的。（常用）**

如果希望记住元素的添加顺序，也没有重复元素需要存储，且希望增加改查都快？
- **用LinkedListHashSet集合（有序、不重复、无索引），底层基于哈希表和双链表。**

如果要对元素进行排序，也没有重复元素需要存储？且希望增加改查都快？
- **用TreeSet集合，基于红黑树实现。**

# Collections

- 是一个用来操作集合的**工具类**

## Collections提供的常用静态方法

| 方法名称                                                                       | 说明                             |
| -------------------------------------------------------------------------- | ------------------------------ |

### 代码
```java
public class CollectionsTest1 {
```

# Map集合体系
![[Pasted image 20260720160807.png]]

**Map本身是一个接口，是不能实例化对象的，需要靠它的实现类来实例对象**

- Map集合称为双列集合，格式：{key1=value1, key2=value2, key3=value3, ...}，一次需要存一对数据作为一个元素。
- Map集合的每个元素 “key=value” 称为一个键值对/键值对象/一个Entry对象，Map集合也被叫做 “键值对集合”
- Map集合的所有键是不允许重复的，但值可以重复，键和值是一一对应的，每一个键只能找到自己对应的值
## 特点

**注意**：Map系列集合的特点都是由键决定的，值只是一个附属品，值是不做要求的
- **HashMap**（由键决定特点）：无序、不重复、无索引； **（用的最多）**
- **LinkedHashMap**（由键决定特点）：由键决定的特点：**有序**、不重复、无索引。
- **TreeMap**（由键决定特点）：**按照大小默认升序排序**、不重复、无索引。

```java
public class Demo2 {
    public static void main(String[] args) {
        Map<String, Double> map = new HashMap<>();

        // put添加元素
        map.put("苹果",2.22);
        map.put("桃子",4.22);
        map.put("西瓜",22.22);
        map.put(null,null);
        System.out.println(map); // {null=null, 桃子=4.22, 苹果=2.22, 西瓜=22.22}

        // size
        System.out.println(map.size()); // 4

        // 判断集合是否为空
        System.out.println(map.isEmpty()); // false

        // 根据key获取元素
        System.out.println(map.get("西瓜")); // 22.22 没有这个key返回null

        // 删除
        map.remove("桃子");
        System.out.println(map); // {null=null, 苹果=2.22, 西瓜=22.22}

        // 是否包含某个键
        System.out.println(map.containsKey("西瓜")); // true

        // 是否包含某个值
        System.out.println(map.containsValue(22)); // false

        // 获取所有key
        Set<String> keys = map.keySet();
        System.out.println(keys); // [null, 苹果, 西瓜]

        // 获取所有值
        Collection<Double> values = map.values();
        System.out.println(values); // [null, 2.22, 22.22]

        // 清空集合
        map.clear();
        System.out.println(map); // {}
    }
}
```

```java
            String key = entry.getKey();
            Double value = entry.getValue();

        }
}
```

## HashMap集合的底层原理

- HashMap跟HashSet的底层原理是一模一样的，都是基于哈希表实现的。

**实际上：原来学的Set系列集合的底层就是基于Map实现的，只是Set集合中的元素只要键数据，不要值数据而已。**

```java
public HashSet() {
    map = new HashMap<>();
}
```

### 哈希表

- JDK8之前，哈希表 = 数组 + 链表  
- JDK8开始，哈希表 = 数组 + 链表 + 红黑树  
- 哈希表是一种增删改查数据，性能都较好的数据结构。
- 如果键存储的是自定义类型的对象，可以通过重写hashCode和equals方法，这样可以保证多个对象内容一样时，HashMap集合就能认为是重复的

## TreeMap

- 特点：不重复、无索引、可排序(按照键的大小默认升序排序，**只能对键排序**)
- 原理：TreeMap跟TreeSet集合的底层原理是一样的，都是基于红黑树实现的排序。


### TreeMap指定排序规则

- 让类实现Comparable接口，重写比较规则。
- TreeMap集合有一个有参数构造器，支持创建Comparator比较器对象，以便用来指定比较规则。
